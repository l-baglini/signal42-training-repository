# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Context

This is a participant folder (`mdand`) inside a Signal42 AI training workshop repository. The challenge for homework-01 is: **build something entirely through AI — no hand-coding allowed**. Any tool, any language, any idea is fair game.

The broader repo lives at `signal42-training-repository/`, organized as:
```
signal42-training-repository/
├── homework-01/
│   ├── mdand/          ← this folder (your work goes here)
│   └── <other-participants>/
└── homework-02/
```

## Contribution Workflow

Work is submitted via **fork + pull request**:

1. Work inside `homework-01/mdand/` — this folder is yours exclusively.
2. Commit and push to your fork.
3. Open a PR from your fork to the main repository.

Commit message convention used in this repo: `homework 1 submission - <Name>` or `homework-01: <name>`.

## What to Build

There is no prescribed stack. Look at sibling folders for inspiration — participants have built Electron apps, CLI tools, web dashboards, games, etc. The only rule: let AI write the code, you direct the thinking.
